# AI Radar Agent v0.2.1 - Feishu Dispatch Stable

This release is the rollback point before the next optimization phase.

## Current Stable Flow

- Feishu automation triggers production at 10:00 Asia/Shanghai.
- Feishu calls GitHub `workflow_dispatch` for `.github/workflows/daily.yml`.
- GitHub native `on.schedule` is disabled to avoid duplicate runs.
- The workflow publishes Feishu docx as the primary reading surface.
- Feishu group notification uses the stable multi-card layout.
- Default recall providers are RSS + Bocha.
- Tavily remains disabled by default to save quota.
- Artifacts are uploaded even when later publish or notification steps fail.

## Feishu Automation Trigger

Use a Feishu automation HTTP step to call:

```text
POST https://api.github.com/repos/<OWNER>/<REPO>/actions/workflows/daily.yml/dispatches
```

Recommended body:

```json
{
  "ref": "main",
  "inputs": {
    "date": "",
    "dry_run": "false",
    "skip_llm": "false",
    "send_bot": "true",
    "output_mode": "feishu_docx_import",
    "tavily_enabled": "false",
    "force_republish": "false",
    "report_lint_policy": "warn",
    "bot_block_on_lint_critical": "false"
  }
}
```

The GitHub fine-grained token should only need the target repository's `Actions: Read and write` permission. Do not store the token in this repository.

## Manual Run Workflow

Manual runs remain available from GitHub Actions:

1. Open GitHub Actions.
2. Choose `Daily AI Radar`.
3. Click `Run workflow`.
4. Leave `date` blank to generate yesterday in Asia/Shanghai.

## Rollback

Use this release as the rollback point:

- Branch: `stable/v0.2.1`
- Tag: `v0.2.1`

If a later change breaks production, restore from this tag or branch and re-run the workflow through Feishu automation or GitHub `Run workflow`.
