## Top Insights

1. Agent-product reliability is framed around workflow clarity, not model novelty alone.
2. Evidence-first operation needs source binding, gate results, and artifact references before narrative.
3. External actions such as document publication, bot notification, or workflow dispatch must remain blocked in demo and eval modes.
4. Run-level observability is strongest when ToolCall records, gate decisions, and artifact refs can be read together.
